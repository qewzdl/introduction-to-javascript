Introduction to Javascript

The History of JavaScript

1. Creation and Initial Development

JavaScript was created in 1995 by Brendan Eich, who developed it in just 10 days for Netscape.

2. Name Evolution

Initially named Mocha, it was later renamed LiveScript and then JavaScript as a marketing strategy to associate it with the popular Java language at the time.

3. Original Purpose

Originally designed for adding simple interactive features to web pages, JavaScript quickly became a cornerstone of web development.

4. Standardization

In 1997, it was standardized by ECMA under the name ECMAScript (ES).

5. Modern Evolution

Since then, the language has evolved significantly, with major updates like ES6 (2015) introducing modern features such as classes, modules, and arrow functions.

6. Current Use

Today, JavaScript is a versatile tool used for building both client-side and server-side applications, making it essential for modern web development.

Interpreted and Compiled Languages

Interpreted languages

In such languages, the code is executed line by line by an interpreter. The program is not pre-translated into machine code; instead, the interpreter reads and executes instructions on the fly. This allows the code to run immediately but can be slower due to line-by-line execution.

Examples:

Python

PHP

Ruby

Compiled languages

In compiled languages, the code is translated into machine code before execution. This process creates an executable file that is then run on the device. Compilation may take time, but the resulting code is usually faster as it is optimized for a specific platform.

Examples:

C

C++

Rust

JavaScript

JavaScript is primarily an interpreted language, but its execution is often accelerated through JIT (Just-In-Time) compilation. Modern engines (such as V8 or SpiderMonkey) first interpret the code and then compile frequently used parts into machine code, improving performance. This makes JS a hybrid between interpretation and compilation.
